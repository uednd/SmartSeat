# SmartSeat 初赛演示脚本

本文档仅记录演示流程，不承载产品需求定义。产品需求以 [PRD.md](PRD.md) 为准。

## 1. 正常预约与使用演示

1. 管理员页面展示座位 A 为空闲、设备在线。
2. 学生账号登录小程序。
3. 学生预约座位 A。
4. 终端屏幕切换为已预约，显示动态二维码。
5. 学生扫码签到。
6. 终端屏幕切换为使用中，状态灯切换为使用中状态。
7. 屏幕显示剩余时间。
8. 学生点击主动离座。
9. 座位恢复空闲，学习记录增加。

## 2. 预约未到演示

1. 学生创建短签到窗口预约。
2. 学生不扫码签到。
3. 后端自动判定预约未到。
4. 座位自动释放。
5. 学生个人页出现预约未到记录。
6. 管理员页面出现预约未到事件。

## 3. 占用异常演示

1. 座位处于空闲。
2. 人靠近或坐到座位区域。
3. 毫米波传感器上报 `PRESENT`。
4. 后端持续判定后生成“疑似未预约占用”异常。
5. 管理员页面展示异常。
6. 管理员标记处理或手动设置维护。

## 4. 提前离座异常演示

1. 学生完成签到，座位处于使用中。
2. 人离开座位区域。
3. 毫米波传感器持续上报 `ABSENT`。
4. 后端生成“疑似提前离座”异常。
5. 学生当前使用页展示提醒。
6. 管理员页面展示异常。

## 5. 设备离线演示

1. 断开终端网络或停止设备心跳。
2. 后端超过 75 秒后判定设备离线。
3. 学生页显示该座位故障/维护，不可预约。
4. 管理员页面显示设备离线异常。
5. 恢复设备网络后，设备重新上线并同步状态。

## 6. 管理员手动释放演示

1. 座位处于使用中或待释放状态。
2. 管理员进入座位管理页。
3. 管理员点击释放座位。
4. 管理员填写释放原因。
5. 后端释放座位并记录操作日志。
6. 终端屏幕和状态灯恢复空闲状态。

## 7. 匿名排行榜演示

1. 系统准备多名测试用户学习记录。
2. 学生进入匿名排行榜页。
3. 页面展示周学习时长榜、到馆次数榜、连续学习榜。
4. 当前学生看到自己的匿名排名。
5. 榜单不展示真实身份。

## 8. 登录模式演示

1. 管理员进入系统配置页。
2. 页面展示当前登录模式。
3. 系统演示微信登录模式下的一键登录。
4. 系统演示 OIDC 模式下通过学校身份源完成真实登录流程。
5. OIDC 模式下不出现注册入口。
