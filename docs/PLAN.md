# SmartSeat 初赛实施计划

本文档仅记录实施计划与阶段安排，不承载产品需求定义。产品需求以 [PRD.md](PRD.md) 为准。

## 第 1–2 周：基础架构与状态机

- 完成需求冻结。
- 完成后端项目初始化。
- 完成 uni-app 小程序项目初始化。
- 完成数据库表设计。
- 完成座位、预约、设备、用户、角色基础模型。
- 完成座位状态机设计与编码。
- 完成 MQTT 服务接入验证。
- 完成真实 OIDC 最小闭环验证：小程序获取登录地址、后端处理回调、签发系统 token、完成角色路由。

## 第 3–4 周：登录、预约与签到闭环

- 完成微信登录模式。
- 完成 OIDC 登录模式正式接入与异常处理。
- 完成角色路由。
- 完成学生座位页。
- 完成预约创建。
- 完成动态二维码 token 生成。
- 完成扫码签到。
- 完成终端屏幕基础显示。

## 第 5–6 周：设备与传感器闭环

- 完成设备 MQTT 心跳上报。
- 完成设备在线/离线判断。
- 完成毫米波传感器适配层。
- 完成传感器数据 MQTT 上报。
- 完成状态灯控制。
- 完成后端向终端下发显示状态和灯光命令。
- 完成设备离线异常。

## 第 7–8 周：管理与异常闭环

- 完成管理员页面。
- 完成座位看板。
- 完成设备管理页。
- 完成异常事件页。
- 完成管理员手动释放。
- 完成预约未到记录。
- 完成空闲占用、提前离座、到期仍有人异常检测。
- 完成管理员操作日志。

## 第 9–10 周：统计、排行榜与演示打磨

- 完成学习记录生成。
- 完成本周到馆次数、累计学习时长、连续学习天数统计。
- 完成匿名排行榜。
- 完成小程序页面视觉整理。
- 完成终端屏幕界面整理。
- 完成演示数据准备。
- 完成答辩演示脚本。
- 完成项目文档和汇报材料。
